# 胎压 (TirePressure) App 端专属页面开发指南

## 背景

当前 `TirePressure.yaml` Spec 已经定义完成，`tslink_spec.ts` 中已注册 `TP` 类型，路由默认指向 `generic-device` 通用页面。本文档描述如何为其创建专属 Page（类似 `TS-CNTR` 的 `CNTR.vue`），提供更好的用户体验。

## 现有基础盘点

在动手之前，先确认以下内容**已经就绪**，无需修改：

| 文件 | 状态 | 说明 |
|------|------|------|
| `specs/TirePressure.yaml` | ✅ 已有 | deviceType: TP，3 个 uplink，0 个 downlink |
| `src/tslink_spec.ts` | ✅ 已注册 | `TP` 条目已存在，CLI 生成 |
| `src/router_map.json` | ⚠️ 需更新 | 当前 `TP` 指向 `generic-device`，需改为专属页面 |
| `pages.json` | ⚠️ 需更新 | 需添加 `pages/devices/TP` 路由条目 |
| `pages/devices/TP.vue` | ❌ 待创建 | 本文档的核心工作 |

### TirePressure Spec 回顾

```yaml
deviceType: TP
fullName: TirePressure
description: 胎压监测设备，支持胎压、温度、电量上报
version: "1.0.0"

uplink:
  - id: 0x01
    name: pressure
    format: u16
    unit: kpa
    description: 胎压值（精度 0.1kPa，需除以 10）

  - id: 0x02
    name: temperature
    format: i16
    unit: celsius
    description: 轮胎温度（精度 0.1°C，需除以 10）

  - id: 0x03
    name: battery
    format: u8
    unit: "%"
    description: 电池电量百分比
```

**与 Counter (CNTR) 的关键差异**：

- TP 是**纯上报型**设备，没有 downlink，App 端只需展示数据，不需要发送控制指令
- TP 有 **3 个上行字段**（CNTR 只有 1 个 count）
- pressure 和 temperature 的原始值需要**除以 10** 才是真实物理量

## 操作步骤

### 第一步：创建 `pages/devices/TP.vue`

在 `host/tslink/pages/devices/` 下创建 `TP.vue`。以下是完整代码，逐段说明。

#### 完整代码

    <DeviceHeader

    <view class="main-content">

        <text class="pressure-unit">kPa</text>
      </view>

        <view class="info-card">
          <text class="info-label">温度</text>

            <text class="info-unit">°C</text>
          </view>
        </view>
        <view class="info-card">
          <text class="info-label">电量</text>
          <view class="info-value-wrapper">
            <text class="info-value">{{ battery }}</text>
            <text class="info-unit">%</text>
          </view>
        </view>
      </view>

      <!-- 胎压状态提示（可选增强） -->
      <view v-if="hasData" :class="['status-tip', pressureStatus]">
        <text class="status-tip-text">{{ pressureStatusText }}</text>
      </view>
    </view>

    <!-- 设备信息 -->
    <view class="footer">
      <text class="device-info">{{ formatMac(mac) }} · {{ spec?.version || 'v1.0.0' }}</text>
    </view>

    <!-- 全局调试组件 -->
    <GlobalDebug :accentColor="accentColor" />
  </view>
</template>

<script>
import { specs } from '@/src/tslink_spec';
import Preferences from '@/src/preferences';
import DeviceHeader from '@/src/components/device-header.vue';
import GlobalDebug from '@/src/components/global-debug.vue';
import { formatMac } from '@/src/ble';

export default {
  components: {
    DeviceHeader,
    GlobalDebug,
  },
  data() {
    return {
      deviceType: 'TP',
      mac: '',
      connection: null,
      // 三个上行数据（原始值）
      pressure: 0,
      temperature: 0,
      battery: 0,
      // 是否已收到过数据
      hasData: false,
      // 偏好设置
      preferences: Preferences.getAll(),
      accentColor: Preferences.getAccentColor(),
    }
  },

  computed: {
    spec() {
      return specs[this.deviceType] || null;
    },

    // 胎压显示值（原始值除以 10，保留一位小数）
    displayPressure() {
      if (!this.hasData) return '--';
      return (this.pressure / 10).toFixed(1);
    },

    // 温度显示值（原始值除以 10，保留一位小数）
    displayTemperature() {
      if (!this.hasData) return '--';
      return (this.temperature / 10).toFixed(1);
    },

    // 胎压状态判断（阈值可根据实际需求调整）
    pressureStatus() {
      const kpa = this.pressure / 10;
      if (kpa < 200) return 'low';
      if (kpa > 300) return 'high';
      return 'normal';
    },

    pressureStatusText() {
      const map = {
        low: '胎压偏低',
        high: '胎压偏高',
        normal: '胎压正常',
      };
      return map[this.pressureStatus] || '';
    },

    // 是否可以与设备交互（虽然 TP 无 downlink，但保留以备扩展）
    canInteract() {
      const state = this.connection?.reactiveState;
      return this.connection?.enabled && state?.physicalState === 'connected';
    },
  },

  onLoad(options) {
    if (options.deviceType) {
      this.deviceType = options.deviceType;
    }
    if (options.mac) {
      this.mac = options.mac;
    }
  },

  onShow() {
    this.preferences = Preferences.getAll();
    this.accentColor = Preferences.getAccentColor();
  },

  methods: {
    formatMac,

    onReady({ connection }) {
      this.connection = connection;
    },

    // 核心：处理上行数据
    onMessage({ data }) {
      data.forEach((item) => {
        switch (item.name) {
          case 'pressure':
            this.pressure = item.value;
            break;
          case 'temperature':
            this.temperature = item.value;
            break;
          case 'battery':
            this.battery = item.value;
            break;
        }
      });
      this.hasData = true;
    },
  }
}
</script>

#### 代码要点说明

**1. 组件复用**

直接复用 `DeviceHeader` 和 `GlobalDebug` 两个公共组件，和 CNTR.vue 完全一致。`DeviceHeader` 负责连接管理（open/enable/disable/close）、状态徽章显示、返回按钮，页面无需关心这些细节。

**2. `onMessage` 数据处理**

这是专属页面的核心。`DeviceHeader` 通过 `@message` 事件把 BLE 上行数据透传过来，格式为：

```js
data = [
  { name: 'pressure', value: 2453 },
  { name: 'temperature', value: 325 },
  { name: 'battery', value: 85 },
]
```

页面按 `name` 分发到对应的 data 字段。注意 TLV→JSON 转换由框架自动完成，页面只需处理 JSON。

**3. 数值转换**

Spec 中声明了 pressure 精度为 0.1kPa、temperature 精度为 0.1°C，所以设备发送的是放大 10 倍的整数值。在 computed 中除以 10 并 `toFixed(1)` 得到显示值。

**4. 胎压状态判断（可选增强）**

CNTR 页面只有一个计数值，没有状态语义。TP 页面可以增加**胎压状态判断**逻辑：根据 pressure 的范围显示"正常/偏低/偏高"提示，配合不同颜色。阈值（200~300 kPa）可根据实际需求调整。

**5. 没有 downlink 操作**

和 CNTR 最大的区别：TP 是纯上报型，没有 `send()` 调用，不需要"重置"按钮或任何控制交互。整个页面只做数据展示。

### 第二步：注册路由

#### 方式 A：运行 CLI 自动更新（推荐）

在 `host/tslink/` 目录下执行：

```bash
python tools/tslink_cli.py -t router -i specs/ -o src/router_map.json
```

CLI 会自动扫描 `pages/devices/` 目录，发现 `TP.vue` 已存在，自动将 `router_map.json` 中的 `TP` 从 `pages/common/generic-device` 更新为 `pages/devices/TP`，并同步更新 `pages.json`。

也可以直接运行 `update_specs.bat`，它会同时执行 ts 和 router 两步生成：

```bash
# Windows
update_specs.bat

# 等价于
python tools/tslink_cli.py -t ts -i specs/ -o src/tslink_spec.ts
python tools/tslink_cli.py -t router -i specs/ -o src/router_map.json
```

#### 方式 B：手动修改（快速验证）

如果想跳过 CLI 直接验证，手动修改两个文件：

**修改 `src/router_map.json`**：

```json
{
  "CNTR": "pages/devices/CNTR",
  "HR": "pages/common/generic-device",
  "LED": "pages/common/generic-device",
  "TEMP": "pages/common/generic-device",
  "TP": "pages/devices/TP"
}
```

**修改 `pages.json`**，在 `pages` 数组中添加一条：

```json
{
  "path": "pages/devices/TP",
  "style": {
    "navigationStyle": "custom"
  }
}
```

### 第三步：验证

1. 在 HBuilderX 中运行项目
2. 确保设备端广播名为 `TS-TP`，且通过 TSLink Service (0xFDE0) 发送 TLV 数据
3. App 扫描到 `TS-TP` 设备后，点击应自动跳转到 `pages/devices/TP` 专属页面
4. 验证胎压、温度、电量三个数据是否正确显示（注意 /10 的精度转换）

**Fallback 保障**：如果 `TP.vue` 文件不存在或路由未更新，框架会自动 fallback 到 `generic-device` 通用页面，不会出错。

## 与 CNTR.vue 对照

| 对比项 | CNTR.vue | TP.vue |
|--------|----------|--------|
| deviceType | `CNTR` | `TP` |
| 上行字段数 | 1 (count) | 3 (pressure, temperature, battery) |
| 下行字段数 | 1 (reset) | 0 |
| 数值转换 | 无 | pressure / 10, temperature / 10 |
| 用户交互 | 重置按钮 + `conn.send()` | 纯展示，无按钮 |
| 页面布局 | 单一大数字居中 | 大数字 + 两个信息卡片 + 状态提示 |
| 公共组件 | DeviceHeader + GlobalDebug | 相同 |
| onMessage 处理 | 单字段 `count` | 多字段 switch 分发 |

---

## 总结

整个工作只涉及 **1 个新文件 + 2 个配置更新**：

1. **创建** `pages/devices/TP.vue` — 专属页面（本文档核心）
2. **更新** `src/router_map.json` — 将 TP 路由指向专属页面
3. **更新** `pages.json` — 注册页面路由条目

其中第 2、3 步通过运行 `update_specs.bat` 即可自动完成。
